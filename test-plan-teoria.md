# 📖 Test Plan — O que é e como criar

> O Test Plan é um dos documentos mais importantes da prática de QA, pois orienta toda a execução dos testes de forma estruturada.

---

## O que é um Test Plan?

Um Test Plan é um documento que descreve **como os testes serão conduzidos**, incluindo escopo, estratégia, objetivos, recursos, cronograma e critérios de sucesso.

Ele responde perguntas fundamentais:

| Pergunta | Responde |
|---|---|
| O que será testado? | Escopo |
| Como será testado? | Estratégia |
| Quando será testado? | Cronograma |
| Quem será responsável? | Recursos humanos |

---

## Tipos de Plano de Teste

### Plano Mestre de Testes
- Documento de alto nível
- Define a estratégia geral do projeto
- Abrange todo o sistema

### Plano de Teste Específico
- Focado em funcionalidades específicas
- Mais detalhado e técnico
- Usado para módulos ou features individuais

---

## Etapas para criar um Test Plan

| # | Etapa | O que define |
|---|---|---|
| 1 | Análise do produto | Entender o que será testado |
| 2 | Estratégia de teste | Como e quais tipos de teste serão aplicados |
| 3 | Definição de objetivos | O que se espera validar |
| 4 | Critérios de teste | Quando suspender e quando encerrar os testes |
| 5 | Planejamento de recursos | Quem e o que será necessário |
| 6 | Ambiente de teste | Onde os testes serão executados |
| 7 | Cronograma e estimativa | Quanto tempo cada etapa levará |
| 8 | Entregáveis | O que será produzido ao final |
